## Steps to Submit

### 1. Fork this Repository

Click the `Fork` button on the top-right side of this repository.

This will create a copy of this repository in your own GitHub account.

### 2. Open Your Forked Repository

After forking, open your copied repository from your GitHub account.

### 3. Open the `submissions` Folder

Inside the repository, open:

```txt
submissions/
```

### 4. Create Your Team Folder

Create a new folder using your team name.

Use simple names without spaces.

Good examples:

```txt
team-alpha
code-warriors
solo-builders
```

### 5. Upload Your Project Files

Upload your complete project files inside your team folder.

Your folder may include:

* Source code
* Frontend files
* Backend files
* Assets
* Documentation
* Screenshots
* Setup files

### 6. Add a README.md File

Every team must add a `README.md` file inside their team folder.

Use this format:

```md
# Project Title

## Team Name
Your Team Name

## Team Members
- Member 1 Name
- Member 2 Name
- Member 3 Name

## Selected Domain
AI Automation / Finance / Education / Manufacturing / Environment & Public Safety

## Problem Statement
Explain the problem your team is solving.

## Solution
Explain your solution in simple words.

## Tech Stack Used
- HTML, CSS, JavaScript
- React / Node.js / Python
- Gemini / Google AI Studio / GitHub

## AI Tools Used
Mention the AI tools used in your project.

## Features
- Feature 1
- Feature 2
- Feature 3

## How to Run the Project
Explain how someone can run your project.

Example:
1. Download the project
2. Open the folder
3. Run the required command
4. Open the project in browser

## Demo / Screenshots
Add demo link or screenshots if available.

## Future Scope
Mention how this project can be improved in the future.
```

### 7. Commit Your Changes

After uploading your files, click `Commit changes`.

Use a commit message like:

```txt
Added project submission for Team Alpha
```

### 8. Create Pull Request

After committing, click:

```txt
Contribute → Open Pull Request
```

### 9. Submit Pull Request

Use this Pull Request title:

```txt
Submission: Team Name
```

In the Pull Request description, add:

```txt
Team Name:
Project Title:
Domain:
Team Members:
```

Then click:

```txt
Create Pull Request
```

---

## Final Submission Checklist

Before the deadline, make sure:

* Team folder is created inside `submissions/`
* Project files are uploaded
* `README.md` is added inside your team folder
* Pull Request is created
* Pull Request title includes your team name
* PPT is uploaded to the Google Drive folder
* All team members have completed SOLO onboarding

---

## Important Rules

* Submit only your own team’s work.
* Do not edit another team’s folder.
* Do not delete files from the main repository.
* Submit before the deadline.
* Late or incomplete submissions may not be considered for evaluation.

---

## Simple Explanation

You are taking a copy of the official repository, adding your team project inside it, and requesting the organizers to accept your submission through a Pull Request.

This helps the organizers track every team’s project, GitHub activity, and final submission properly.
